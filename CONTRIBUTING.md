# Contributing（運用メモ）

このリポジトリは学習用です。厳密なルールよりも「**増やしやすさ**」を優先します。

## トピックの追加方法

推奨: テンプレから作る

```bash
./scripts/new-topic.sh "<topic-name>"
```

`<topic-name>` は以下を推奨します。

- **kebab-case**（例: `http-cache`, `jwt`, `react-server-components`）
- 日本語でも構いませんが、パス・URL共有の都合で英数字が無難です

## 各トピックの書き方（目安）

`topics/<topic-name>/README.md` に、最低限これだけあると後で効きます。

- **What / Why**: 何で、何が嬉しい/困っているか
- **結論（先に）**: 今の理解の要点を箇条書きで
- **検証メモ**: `notes.md` に時系列で残す（失敗ログも価値）
- **links**: 公式ドキュメント、仕様、PR、Issue、ブログ等
- **TODO**: 次に読む/試すこと

## コードとテストについて

- `code/` は **最小再現** を意識（依存と手順を小さく）
- `tests/` は「学んだことを固定」するために追加（言語/フレームワークは自由）
- 実行手順は各トピックの `README.md` に書く

## トピック一覧の更新

`topics/README.md` の一覧に追加してください（後で探すための索引になります）。
